The warning message below appears when, as an administrator, you choose to modify the app config settings.  When selected, the first screen displayed is a warning. The warning states that great care should be used when editing the Application Configuration File as changes could disable the application.

The next screen displayed is the editor. This screen will also be displayed during updates to the application as it allows an administrator to replace and then update the application configuration file quickly and easily. This screen also provides the admin or user with the ability to backup their current application configuration file so that a restore or fall back can be accomplished simply by pushing the purple button, “Restore App Config File”.

First Click on Step 1, as indicated, to identify your network Servers.  This may take a minute and will search the network for available servers.  Next pull down the list using the arrow under “Repository Server”.  You will need to select the server with which you are working, the one that holds the ECM Library repository, from the displayed list.

   Next  #2 is optional – get databases.  It helps verify your connections for trouble shooting purposes.

 

Next , you can press #3  the Edit App Config button. This will open the app config file and display the contents within the edit screen. Changes can be made if ABSOLUTELY necessary – otherwise, do not change the app config file. This screen is intended to make the transition from our servers to yours very simple and easy. Once editing the app config file, you can press the #4  Auto-Apply Changes and the ECM Library default server, SP6000, will be automatically replaced with the name of the server displayed in the servers drop down box. Once the changes have been correctly applied and verified, the #5 Save Changes button can be pressed and the app config file will be replaced. The application must be restarted for the changes to take effect.

Next Press the #6  Backup App Config File button and the newly changed file will be set as the backup/replacement app config file. This will be very useful when applying updates to the application as received from the ECM Library development team.
