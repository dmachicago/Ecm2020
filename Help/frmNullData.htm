Dynamic Data Display

(frmNullData)

 

The original intent of this screen was to assist administrators in finding content that exists in the repository and has no viable data within the records. This can be caused for any number of reasons, but most common is if the document/content selected for load is corrupt or locked. In this case, a place marker is entered into the repository and a NULL entry is placed in the binary content field. Press the Show NULL Content button to display a list of all source content in the repository that has a null entry for the binary image. Press the DELETE button to remove these entries from the repository. You can take this list of entries, find the associated content on the specific computer and determine why the content is not loading.

This screen was expanded to allow administrators to execute SQL directly against the repository. It is not designed to replace the SQL Manager, but instead, it is here to give quick and easy access to the repository via a full SQL interface.

To access the SQL editor, check the Show SQL Dialog check box and the editor will be displayed. Fill in the SQL you wish to execute and press the Execute SQL button.
