Information on the Quick Search Screen

This screen provides the user with the ability to perform searches on both emails and content (documents, spreadsheets, graphic files, etc.) quickly and easily. The level of granularity is limited unlike the much more powerful search screens provided for detailed extractions of emails and documents/content. You will find that the majority of searches can be satisfied using this one screen. When the need arises to drill down into a more sophisticated search, please use the detailed search screens for Email or Content.


Quick  Search Purpose:

Quick Search provides the capability to quickly and easily find content and emails within the repository. To open the Search Assistant, place your cursor inside the Search Command text line and press the right mouse key. This will open the Search Assistant.

Screen Components:

The Search Line:                               Enter your search into this line and press search or enter.  The easiest way to get started is to enter the words you are interested in and press enter or the Search button. To include all the words, use a “+” sign in front of them or separate your words with blanks to include any of the listed words. It is that simple to get started.

Retrieval Options:                           Once you have a returned search and clicked and highlighted the file(s) you wish to use for later, you can preview files, restore the files to a directory, or you can add them to pre-defined library. 

Preview: To preview files, double click on a row in the returned search grid list and it will open up that file using the default application used by your computer.

Restored Content Directory:  Press this button to go to the directory containing the restored content. The location of this directory can be changed using the user parameters screen.

Restored Emails Directory: Press this button to go to the directory containing the restored emails. The location of this directory can be changed using the user parameters screen.

Add to Library: First select a library from the pull down menu to the left of “Add to Library”.  Highlight the row or rows of files you wish to add to the library.  Click on “Add to Library”.  A pop-up menu will verify your selection. Click OK.  You and library members can then review these files going forward using the Library Items screen in ECM Library.                       

Type Search                                       You can use this to search across the repository for only emails, or only content, or you can search on both at the same time.

Show Details                                     This will open the details screen, a floating screen, that shows the associated metadata for a currently selected email or content file.  This will open up details to a full screen.  You can minimize this screen by clicking on the window box in the top upper right of the window.  The screen will then remain in a smaller size.  From here you can scroll up and down a row of returned items on one screen while viewing the details in this details screen.  When you are finished, click on the x in the upper right hand part of the window.

Business Meaning                           This provides a quick way to broadly search across content and emails, as each search word you enter will include an inflection on that word and will join it with an “OR” statement for all words.  Caution: this search will run slower due to its expanded search nature.  Please refer to this link.

Show Weights                                  Weights help identify the relevancy of your content results to your search query.  Search results are sorted with the top line showing the highest weight ranking.   For more information,  Please refer to this link.

GEN SQL                                              There are many times when it is of great advantage to be able to see the actual SQL statement that is being produced for any given search. When the GEN SQL button is pushed, the same SQL that is used to extract the search is produced and placed into the clip board.

                                                                This allows you, the user, to view the SQL in an external tool or, on occasion, produce and send ECM Library support the actual SQL query that may be producing issues.
